# BuildFlow Roadmap

This roadmap covers BuildFlow Local only. It tracks the free, open-source, self-hosted product surface.

## Current focus

- dashboard polish
- local activity feed UI
- local source management
- safer write-policy UX
- Custom GPT setup clarity
- local verification and troubleshooting
- screenshots and demo assets
- onboarding and first-run improvements
- context, search, and read improvements
- execution packet UX
- open-source contribution flow

## Near-term priorities

- make the dashboard easier to scan at a glance
- improve source activation and reindex flows
- keep activity feedback concise and user-facing
- make safe writes and preflight checks easier to understand
- reduce first-run setup friction for new GitHub visitors

## Safety and scope

- keep secrets, private keys, traversal paths, and protected files blocked
- keep `dryRun` / `preflight` as no-write checks
- keep verified writes explicit and visible
- keep confirmation-gated maintenance operations guarded
