# BuildFlow v1.2.0-beta

## Status
BuildFlow Local is the public GitHub beta target. BuildFlow Managed is later and out of scope unless it directly blocks the Local beta.

## What v1.2.0-beta includes
- Local-first planning and handoff workflow
- Local dashboard
- Local source management
- Custom GPT setup for self-hosted users
- Copy-ready Codex and Claude Code handoff prompts
- Fresh-clone verification guidance
- GitHub issue templates and contribution guidance

## What remains beta
- Dashboard polish
- Launch assets and screenshots
- Additional onboarding clarity
- Any remaining docs friction found by fresh-clone testing

## Free GitHub scope
- Fully self-hosted BuildFlow Local
- User-owned endpoint or tunnel if ChatGPT needs callback access
- No BuildFlow Managed dependency for the free beta

## Not included
- Hosted SaaS account
- Billing
- Team workspaces
- Managed execution sessions
- BuildFlow Managed as the default path

## Verification evidence
- Public beta gate and docs now describe the Local-first path clearly.
- Fresh-clone verification guidance is documented to keep the current runtime untouched.
- GitHub issue templates and a Local-first contributing guide are in place.

## Known limitations
- The beta still depends on the user following the documented Local setup.
- Managed/SaaS is a future path, not part of the free GitHub release.

## Notes
- This release note is intentionally concise so it can be expanded when the beta is fully verified.
